Tu construis un bot backend Node.js pour NF Society.

Objectif :
Créer un système de paiement automatique sécurisé fonctionnant sur Replit,
en utilisant un Gnosis Safe pour gérer la trésorerie NF Society.

IMPORTANT :
- Il n’y a PAS encore de module d’automation installé sur le Safe.
- Le projet doit inclure les instructions pour configurer ou intégrer un module d’automation Safe.
- Le bot ne doit stocker AUCUNE clé privée en production.
- Seul le Safe (via module ou transaction validée) peut exécuter les transferts ERC20.
- Le token utilisé pour les paiements est le NF-CRC wrappé en ERC20 sur Gnosis Chain.

Responsabilités du bot :

1. Écouter les événements de gagnants :
   - Depuis une API backend
   - Ou depuis des événements d’un smart contract
   - Données reçues :
        winnerAddress
        amountToPay
        gameId

2. Vérifier et sécuriser :
   - Empêcher les paiements en double pour un même gameId
   - Vérifier que amountToPay respecte une limite définie
   - Vérifier que le Safe possède assez de fonds
   - Logger toutes les validations

3. Gestion du Safe :
   - Si aucun module d’automation n’est installé :
        → proposer soit :
            a) création d’un module custom
            b) exécution via transaction Safe classique
   - Fournir les étapes pour installer/configurer un module d’automation Safe
   - Prévoir une architecture évolutive vers module sécurisé

4. Exécution du payout :
   - Token : ERC20 NF-CRC
   - Destinataire : winnerAddress
   - Montant : amountToPay
   - Transaction exécutée via Safe (module ou transaction proposée)

5. Logs :
   - winnerAddress
   - amount
   - gameId
   - transactionHash
   - timestamp
   - status (success / failed)
   - retry automatique (max 3 fois)

Contraintes de sécurité :
- Pas de clé privée en clair dans le code
- Utiliser variables d’environnement si nécessaire
- Prévoir séparation entre environnement test et production
- Toutes les opérations doivent être auditables

Optionnel :
- Wrapper NF-CRC en ERC20 au moment du payout si nécessaire
- Ajouter monitoring ou alertes
- Structurer le projet pour pouvoir ajouter un module d’autation plus tard

Stack technique :
- Node.js
- ethers.js
- Safe Protocol Kit
- Gnosis Chain RPC
- JSON ou petite base pour logs

Objectif final :
Créer un bot de paiement automatique sécurisé, scalable,
avec une architecture prête pour intégrer un module d’automation Safe.
