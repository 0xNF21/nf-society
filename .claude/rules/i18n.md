# i18n Rules

- Toute string visible par l'utilisateur DOIT etre dans `translations` de `src/lib/i18n.ts`
- Jamais de texte en dur dans le JSX (sauf emojis)
- Chaque cle a obligatoirement `fr` ET `en`
- Utiliser `const { locale } = useLocale()` puis `t.maClé[locale]`
- Les traductions sont organisees par section : `morpion`, `memory`, `shop`, `daily`, `demo`, etc.
- Avant d'utiliser `t.xxx[locale]`, verifier que `xxx` existe dans la bonne section de translations
- Erreur frequente : utiliser une cle de `translations.shop` dans un composant qui utilise `translations.daily`
